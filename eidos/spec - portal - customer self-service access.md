# Spec: Customer Portal — Self-Service Invoice Access

## Purpose

A lightweight public-facing portal allowing customers to view their own invoices without requiring an internal VVS account. Access is granted via magic link (token-in-URL), generated by an admin.

## Auth Model

**Token-as-session:** No separate session table. The portal token IS the session credential.

1. Admin clicks "Send portal access" on customer detail → system generates `PortalToken`, returns URL
2. Customer opens `/portal/auth?token=<plain>` → SHA-256(plain) → `FindByHash` → not expired → set `vvs_portal` cookie (value = plain) → redirect `/portal/invoices`
3. On every portal request: middleware reads `vvs_portal` cookie → SHA-256 → `FindByHash` → extracts `CustomerID` → injects into context
4. Token expired or invalid → redirect to `/portal/auth?expired=1`

### Token properties
- 32-byte CSPRNG → base64url → SHA-256 stored (same as InvoiceToken)
- Default TTL: 24h (admin-generated), 7d (dunning-auto)
- `portal_tokens` table, ON DELETE CASCADE from customers

## Route Map

All portal routes registered via `PublicModuleRoutes` **before** `RequireAuth` middleware.

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/portal/auth` | none | Validate token, set cookie, redirect |
| GET | `/portal/invoices` | portal | Invoice list for authenticated customer |
| GET | `/portal/invoices/{id}` | portal | Invoice detail + PDF download link |
| POST | `/portal/logout` | none | Clear `vvs_portal` cookie |
| POST | `/api/customers/{id}/portal-link` | admin | Generate token, return URL |

## Security

- Cookie: `vvs_portal`, `HttpOnly`, `SameSite=Lax`, `Secure` when configured
- Every request validates token freshness — no stale sessions
- Invoice ownership verified: `invoice.CustomerID == portalCustomerID`
- No cross-customer data leakage possible (customerID from token, not URL)
- `Cache-Control: no-store` on all portal responses

## Data Access (read-only)

Customer can only see:
- Their own invoice list (number, date, amount, status)
- Invoice detail (line items, totals)
- PDF link (generates fresh `invoice_token` per visit, 48h TTL)

## UI

- `PortalLayout`: no admin sidebar, no NATS, no notifications widget
- Header: "VVS Customer Portal" + customer name + logout button
- Tailwind + dark theme consistent with admin UI
- No Datastar required (static server-rendered pages)
